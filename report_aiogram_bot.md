# Звіт про чат-бот на Python з використанням бібліотеки Aiogram 3.0

## Вступ

У рамках проекту було розроблено чат-бот для Telegram з використанням бібліотеки Aiogram 3.0. Бот призначений для адміністрування інтернет-магазину одягу, надаючи користувачам можливість переглядати каталог товарів, додавати їх до кошика, оформлювати замовлення та здійснювати оплату через Telegram Payments API. Бот також включає функціонал для адміністраторів.

## Ініціалізація проекту

- **Встановлення бібліотеки**: Aiogram 3.0 встановлено у віртуальному середовищі Python.
- **API Token**: Отримано через BotFather у Telegram.
- **Структура проекту**: Код організовано у директорії `bot_aiogram/` з модулями для обробників, клавіатур, станів тощо.

## Створення базової структури бота

- **Команда /start**: Надсилає привітання та інструкції користувачу.
- **Інструкції**: Список доступних команд після запуску.

## Реалізація команд

- **/help**: Список команд з описом.
- **/info**: Інформація про бота та його функціонал.
- **/catalog**: Перелік доступних товарів.

## Реалізація каталогу товарів

- **Команда /catalog**: Показує список товарів з інлайн-кнопками для перегляду деталей.
- **Кнопка "Замовити"**: Підтвердження замовлення при натисканні.

## Оформлення замовлень

- **Команда /order**: Оформлення замовлень через інлайн-кнопки.
- **Валідація**: Перевірка даних (ім'я, телефон, адреса).
- **Відправка адміністраторам**: Замовлення надсилаються у приватні повідомлення.

## Адміністрування

- **Команда /admin**: Доступ до панелі адміністратора.
- **/add_item**: Додавання товарів до каталогу.
- **/remove_item**: Видалення товарів.
- **/orders**: Перегляд та підтвердження/відхилення замовлень.

## Використання фільтрів та Router

- **Router**: Розділення логіки користувача та адміністратора.
- **Фільтри**: Перевірка ролі користувача (адміністратор чи користувач).

## Створення інтерактивних клавіатур

- **Reply Keyboard**: Для основних команд (/catalog, /info, /help).
- **Inline Keyboard**: Для товарів у каталозі (перегляд, замовлення).

## Логування

- **Система логування**: Запис дій користувачів та адміністраторів у файл `bot.log`.

## Додаткові завдання

- **Збереження даних**: SQLite база даних для товарів та замовлень.
- **FSM**: Finite State Machine для складних операцій (оформлення замовлення).
- **Інтеграція з платіжною системою**: Telegram Payments API для оплати.
- **Оповіщення**: Повідомлення адміністраторам про нові замовлення.

## Основні функції бота

### Каталог товарів
- Перегляд категорій, деталей товарів (назва, опис, ціна, наявність).
- Додавання до кошика.

### Кошик
- Перегляд, редагування (видалення), оформлення замовлення.

### Оформлення замовлення
- Введення даних користувача, вибір доставки, скасування.

### Система оплати
- Інтеграція з Telegram Payments, підтримка провайдерів (Stripe, LiqPay).
- Вибір валюти, статус оплати.

### Користувацький інтерфейс
- Інтерактивні кнопки, меню для каталогу та кошика.

## Інтеграція системи оплати

- **Платіжні функції**: Генерація рахунків, вибір доставки, підтвердження платежу.
- **Структура оплати**: Оплата після підтвердження замовлення.
- **Налаштування**: Функції для обробки платежів у Aiogram 3.0.

## Завантаження на сервер

- **Сервер**: VPS (DigitalOcean, AWS).
- **Налаштування**: Встановлення Python, залежностей, запуск бота, systemd для автозапуску.

## Тестування та підтримка

- **Тестування**: Перевірка всіх функцій, обробка помилок.
- **Підтримка**: Моніторинг, оновлення.

## Висновки

Бот повністю функціональний, з інтерактивною клавіатурою, адмініструванням та інтеграцією оплати. Код чистий, використовує роутери та фільтри. Готовий до розгортання на сервері.

## Документація

Код у `bot_aiogram/`:
- `bot.py`: Головний файл.
- `handlers/`: Обробники для користувачів, каталогу, кошика тощо.
- `keyboards/`: Клавіатури.
- `states/`: FSM стани.
- `middlewares/`: Логування.
- `database.py`: Робота з БД.
